# Система управления каналами

## Обзор

Теперь парсер получает список каналов из базы данных через API бэкенда, вместо жестко заданных переменных окружения. Это позволяет управлять каналами через веб-интерфейс.

## Новые переменные окружения для парсера

Добавьте следующие переменные в `.env` файл парсера:

```env
# Backend API connection
API_BASE_URL=http://localhost:3001
API_USERNAME=your_backend_username
API_PASSWORD=your_backend_password
```

## API Endpoints для управления каналами

### Получить все каналы
```
GET /channels
Authorization: Bearer <token>
```

### Создать новый канал
```
POST /channels
Authorization: Bearer <token>
Content-Type: application/json

{
  "username": "@channel_name",
  "channel_id": 1234567890
}
```

### Получить канал по ID
```
GET /channels/:id
Authorization: Bearer <token>
```

### Удалить канал
```
DELETE /channels/:id
Authorization: Bearer <token>
```

### Получить ID каналов для парсера
```
GET /channels/parser/ids
Authorization: Bearer <token>
```

## Структура данных канала

```typescript
interface Channel {
  id: string;
  username: string;       // @channel_name
  channel_id: number;     // Telegram channel ID
  created_at: Date;
}
```

## Как это работает

1. **Инициализация**: При запуске парсер подключается к API и получает список каналов
2. **Аутентификация**: Используется логин/пароль для получения JWT токена
3. **Периодические обновления**: Каждые 5 минут парсер обновляет список каналов
4. **Отказоустойчивость**: При ошибке подключения к API парсер повторяет попытку через 30 секунд

## Настройка

1. Убедитесь, что бэкенд запущен и доступен
2. Создайте пользователя в системе через API `/auth/register`
3. Добавьте переменные окружения API в парсер
4. Добавьте каналы через API или веб-интерфейс
5. Запустите парсер

## Преимущества

- ✅ Динамическое управление каналами без перезапуска парсера
- ✅ Централизованное хранение настроек в базе данных
- ✅ Возможность управления через веб-интерфейс
- ✅ Автоматическое обновление списка каналов
- ✅ Отказоустойчивость при проблемах с API
